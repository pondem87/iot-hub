# Use case 1: Managing accounts

## use case 1.1: Register new account

#### 1. Intent
- Add a new account to the system under which users, roles, assets etc can be managed

#### 2. Primary Actor
- Account administrator (human owner)

#### 3. Aggregate root
- Account

#### 4. Preconditions
- valid account name - must be globally unique
- admin phone number is valid

#### 5. Command/Inputs
- CreateAccountCommand
    - phone number
    - account name
    - admin's name
    - password

#### 6. State changes
- new account created
- first user created
- user verification pending
- verification code generated

#### 7. Invariants
- each account must have at least 1 user (the admin)
- account name is unique
- founder must be active if organisation is active

#### 8. Domain Events
- AccountCreatedEvent
- UserCreatedEvent

#### 9. Side effects
- create default roles
- send verification code
- send welcome message
- initialise trial subscription

#### 10. Failure modes
- AccountNameAreadyExists
- InvalidUserIdentity
- InvalidPasswordFormat
- AccountCreationLimitExceeded

#### 11. Variants
scenario B: User already exists
- user must be logged in
- additional failure mode: UserNotAuthenticated
