# Use case 1: Manage user account

## use case 2.1: Create new account

#### 1. Intent
- Add a new user to a hub account

#### 2. Primary Actor
- User account owner (human owner)

#### 3. Aggregate root
- Account/User

#### 4. Preconditions
- valid user name
- valid user phone number is valid
- valid password
- valid invitation to an account

#### 5. Command/Inputs
- JoinHubAccountCommand
    - phone number
    - user name
    - password
    - invitation id

#### 6. State changes
- new user created
- invitation accepted
- user verification pending
- verification code generated

#### 7. Invariants
- user must not already exist
- userId is unique and valid phone number
- hub account must be active

#### 8. Domain Events
- InvitationAcceptedEvent
- UserCreatedEvent

#### 9. Side effects
- create default roles
- send verification code
- send welcome message

#### 10. Failure modes
- UserAlreadyExists
- InvalidUserIdentity
- InvalidPasswordFormat
- InvalidInvitation
- AccountNotActive

#### 11. Variants

## use case 2.2: Accept account invitation

#### 1. Intent
- Add user to a hub account

#### 2. Primary Actor
- User account owner (human owner)

#### 3. Aggregate root
- Account/User

#### 4. Preconditions
- valid user must be logged in
- invitation must exist, not be cancelled and match current userId

#### 5. Command/Inputs
- AcceptHubAccountInvitationCommand
    - invitation id

#### 6. State changes
- invitation accepted
- user added to hub account

#### 7. Invariants
- hub account must be active

#### 8. Domain Events
- InvitationAcceptedEvent

#### 9. Side effects
- add default roles

#### 10. Failure modes
- InvalidInvitationId
- AccountNotActive

#### 11. Variants

---

## use case 2.3: Invite user to join hub account

#### 1. Intent
- Invite user to a hub account

#### 2. Primary Actor
- Hub account admin or user with create invitation permission

#### 3. Aggregate root
- Account/Invitation

#### 4. Preconditions
- valid user must be logged in
- hub account must be active
- invitation must not already exist or must be cancelled if it exists

#### 5. Command/Inputs
- InviteUserToAccountCommand
    - valid user phone number

#### 6. State changes
- invitation created

#### 7. Invariants
- hub account must be active

#### 8. Domain Events
- InvitationCreatedEvent

#### 9. Side effects
- invitation notification sent to invited user

#### 10. Failure modes
- InvalidUserIdentity
- AccountNotActive
- InvitationAlreadyAccepted

#### 11. Variants
if invitation already exists
  - resend notification
if invitation exists and is cancelled or revoked
  - reactivate invitation
  - resend notification

---

## use case 2.4: Cancel account invitation

#### 1. Intent
- Cancel user invitation to a hub account

#### 2. Primary Actor
- Hub account admin or user with write permission for invitation

#### 3. Aggregate root
- Account/Invitation

#### 4. Preconditions
- must be logged in as admin or user with correct permission
- invitation must exist and not be cancelled

#### 5. Command/Inputs
- CancelHubAccountInvitationCommand
    - invitation id

#### 6. State changes
- invitation cancelled

#### 7. Invariants
- hub account must be active
- invitation must not be already accepted

#### 8. Domain Events
- InvitationCancelledEvent

#### 9. Side effects
- invitation cancellation notification sent

#### 10. Failure modes
- InvalidInvitationId
- AccountNotActive
- InvitationAlreadyAccepted
- InvalidInvitation

#### 11. Variants
